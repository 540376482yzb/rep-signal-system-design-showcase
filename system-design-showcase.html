5-minute design review

Design a simple Rep Signal pipeline.

Initial scale is about 100 users, each with roughly 9 market rows. That is 900 signal extraction units per curation run.

Intention Offload batch coordination to GitHub Action and keep the Node API layer thin, synchronous, and workerless.
Curate Find signals per profile market row
Send Email one selected lead
Access Dashboard + domain signal APIs

DB And Module Design

Postgres owns profile and send history. Recommendation logic stays in modules and depends on the package/cache boundary.

rep_signal_profiles

Durable user profile, settings, cadence, and market/channel selections.

profile_id email market_keys channel_keys feed_config_json cadence

rep_signal_email_sends

Durable send intent, provider result, selected signal snapshot, and dedupe history.

profile_id status dedupe_key signal_json message_id sent_at

Signal Extractor

Shared by curation and client reads. Converts profile + market row into signal cards.

getRepSignalStrataPackage

Single package dependency for raw menu, mention, and sales strata.

Email delivery module

Receives one selected lead and updates send history around provider calls.

HTTP Endpoints

One internal curation surface for GitHub Action, plus two client read surfaces for gs-app.

GET

/rep-signal/markets

Return users keyed by profile with market_rows.

Input cron auth
Response { users: { [profile_id]: { email, market_rows } } }
POST

/rep-signal/curate

Curate one { profile_id, market_row }.

Input { profile_id, market_row: { market, channel } }
Response { profile_id, market_row, signals: { social, menu, sales } }
POST

/rep-signal/email-sends

Send the selected lead with intent-first history.

Input { profile_id, signal, market_row }
Response { send_id, status, message_id? }
GET

/rep-signal/signals/dashboard

Return all domains, capped at 3 signals per domain.

Input user auth
Response { signals: { social: [3], menu: [3], sales: [3] } }
GET

/rep-signal/signals/:domain

Return 8 signals for one configured domain.

Input domain = social | menu | sales
Response { domain, signals: [8], warnings? }
